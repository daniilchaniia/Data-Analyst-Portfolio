# Ecommerce Funnel & Traffic Analytics

## Описание проекта

Проект посвящен анализу ecommerce-воронки и эффективности источников трафика для сервиса онлайн-продаж.

Цель анализа — оценить, как пользователи проходят путь от первого просмотра страницы товара до покупки, на каких этапах происходит потеря пользователей, какие каналы трафика дают более качественную аудиторию и как поведение пользователей связано с итоговой выручкой.

Анализ построен на событийной таблице `user_events`, где фиксируются действия пользователей на сайте: просмотр страницы товара, добавление товара в корзину, начало оформления заказа, ввод платежной информации и покупка.

## Бизнес-задача

Для ecommerce-продукта важно понимать не только общий объем трафика, но и качество этого трафика: сколько пользователей доходит до покупки, на каком этапе они отваливаются и какие каналы привлечения приводят наиболее конверсионную аудиторию.

В рамках проекта решались следующие вопросы:

- сколько пользователей проходит каждый этап воронки;
- какая конверсия между соседними этапами;
- какая общая конверсия из просмотра товара в покупку;
- какие источники трафика дают больше просмотров;
- какие источники трафика лучше конвертируют пользователей в корзину и покупку;
- сколько времени в среднем проходит между просмотром товара, добавлением в корзину и покупкой;
- какая выручка приходится на заказ, покупателя и посетителя.

## Данные

Основная таблица: `user_events`.

Ключевые поля:

- `user_id` — идентификатор пользователя;
- `event_type` — тип события;
- `event_date` — дата и время события;
- `traffic_source` — источник трафика;
- `amount` — сумма покупки для событий типа `purchase`.

Используемые этапы воронки:

1. `page_view` — просмотр страницы товара;
2. `add_to_cart` — добавление товара в корзину;
3. `checkout_start` — начало оформления заказа;
4. `payment_info` — ввод платежной информации;
5. `purchase` — покупка.

## Этапы анализа

### 1. Расчет количества событий на каждом этапе воронки

На первом этапе рассчитано общее количество событий для каждого шага ecommerce-воронки:

- просмотры страницы товара;
- добавления в корзину;
- начало оформления заказа;
- ввод платежной информации;
- покупки.

Это позволяет оценить общий объем активности пользователей на каждом этапе.

### 2. Расчет уникальных пользователей по этапам

Дополнительно рассчитано количество уникальных пользователей на каждом этапе. Это более корректный показатель для анализа продуктовой воронки, так как один пользователь может совершить несколько событий одного типа.

Для этого использовалась логика `COUNT(DISTINCT CASE WHEN ...)`.

### 3. Конверсия между этапами воронки

Для оценки эффективности пользовательского пути рассчитаны conversion rate между соседними этапами:

- просмотр страницы → добавление в корзину;
- добавление в корзину → оформление заказа;
- оформление заказа → ввод платежной информации;
- ввод платежной информации → покупка;
- просмотр страницы → покупка.

Эта часть анализа помогает определить, на каком этапе пользовательский путь теряет больше всего потенциальных покупателей.

### 4. Анализ источников трафика

Проект также включает анализ источников трафика:

- `organic`;
- `email`;
- `social`;
- `paid_ads`.

Для каждого источника рассчитана доля в общем количестве просмотров страницы товара. Это позволяет понять, какие каналы дают основной объем входящего трафика.

### 5. Конверсия по источникам трафика

Помимо объема трафика, рассчитана эффективность каналов с точки зрения конверсии:

- просмотры страницы;
- добавления в корзину;
- покупки;
- конверсия из просмотра в корзину;
- конверсия из корзины в покупку.

Такой анализ помогает отделить каналы, которые просто приводят много пользователей, от каналов, которые приводят пользователей с высокой покупательской готовностью.

### 6. Анализ времени прохождения воронки

Для пользователей, совершивших покупку, рассчитано среднее время между ключевыми этапами:

- от просмотра товара до добавления в корзину;
- от добавления в корзину до покупки;
- от просмотра товара до покупки.

Этот анализ позволяет оценить скорость принятия решения пользователем и выявить возможные задержки в пользовательском пути.

### 7. Анализ выручки

В финальной части рассчитаны revenue-метрики:

- общее количество посетителей;
- количество покупателей;
- общая выручка;
- количество заказов;
- средняя выручка на заказ;
- средняя выручка на покупателя;
- средняя выручка на посетителя.

Эти показатели позволяют связать продуктовую воронку не только с пользовательским поведением, но и с финансовым результатом.

## Основные метрики

- количество событий на каждом этапе воронки;
- количество уникальных пользователей на каждом этапе;
- конверсия между этапами;
- общая конверсия в покупку;
- распределение трафика по каналам;
- конверсия по источникам трафика;
- среднее время прохождения воронки;
- revenue per order;
- revenue per buyer;
- revenue per visitor.

## Визуализация

Результаты анализа подготовлены для визуализации в Power BI.

Дашборд фокусируется на следующих блоках:

- ecommerce-воронка по этапам;
- конверсия между этапами;
- сравнение источников трафика;
- карточки с revenue-метриками;
- показатели выручки на заказ, покупателя и посетителя.

## Результат проекта

В результате проекта была построена аналитическая основа для оценки эффективности ecommerce-воронки и каналов привлечения.

Проект показывает умение:

- работать с событийными данными;
- строить SQL-запросы для продуктовой аналитики;
- рассчитывать conversion rate и revenue-метрики;
- анализировать эффективность источников трафика;
- готовить данные для BI-дашборда;
- переводить технический SQL-анализ в понятные бизнес-выводы.

## Возможные бизнес-выводы

На основе такого анализа бизнес может:

- определить этапы воронки с наибольшими потерями пользователей;
- сравнить качество разных источников трафика;
- перераспределить маркетинговый бюджет в пользу более конверсионных каналов;
- улучшить этапы оформления заказа и оплаты;
- отслеживать влияние изменений сайта на конверсию и выручку.
